# Squadra - Sistema de Gestión de Mantenimiento

¡Bienvenido al repositorio de Squadra! 

Si eres un **humano (desarrollador)** ingresando al proyecto, estos son los pasos para comenzar:
1. Asegúrate de tener Node.js instalado (v18 o superior).
2. Clona este repositorio y corre `npm install` en la terminal.
3. Copia el archivo `.env.example` (cuando lo generemos) a `.env` y coloca las credenciales de tu base de datos de pruebas.
4. Tu fuente de verdad es siempre la pestaña de [Issues de GitHub](https://github.com/ValeCuatro/Squadra/issues). Reclama un Issue y a trabajar.

Si eres un **Modelo de IA** asistiendo a un desarrollador en este proyecto, DEBES empezar leyendo este contexto obligatoriamente en este orden:

1. Lee detalladamente la especificación principal de negocio ubicada en `SPECS.md` en la raíz.
2. Lee las reglas estrictas de flujo de GitHub que NO PUEDES ROMPER ubicadas en `.antigravity/rules/RULE_GIT.md`.
3. Evita inventar o hardcodear código asimilando las reglas ubicadas en `.antigravity/rules/RULE_INTEGRITY.md`.
4. Asimila los de lineamientos de Arquitectura, Base de Datos y Backend ubicados en la carpeta `.antigravity/skills/` (`SKILL_DB.md`, `SKILL_APP.md`, `SKILL_BACKEND.md`).
5. Solicítale al humano que asigne y vincule el siguiente Issue a resolver usando el MCP de Github (los Issues dictarán tu próximo micro-objetivo). No vayas por libre.

¡Fin de la inyección de contexto! Procede con las peticiones del usuario.
